# Link, Script & Image Components in Next.js

<br>

## `Link` Component - The better version of `<a>` tag

In normal web apps, we use `<a>` tag for navigating pages.  
But, it reloads the page while navigating.  
`<Link>` tag is much better because it doesn't reload the page.

See this example:  
There is a page which has a Navbar. Even if you navigate to other page, the app won't reload.  
We can make the Navbar be available for every page by adding the Navbar component in the `layout.js`. Now, no matter what page you are in, the Navbar will always be there.

`Navbar.js`
```javascript
import Link from "next/link"

const Navbar = () => {
  return (<>
    <ul className="p-5 flex gap-10 text-xl">
      <Link href="/">Home</Link>
      <Link href="/about">About</Link>
      <Link href="/contact">Contact</Link>
    </ul>
  </>)
}

export default Navbar
```

<br>

`layout.js`
```javascript
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";
import Navbar from "./components/Navbar";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata = {
  title: "Next App",
  description: "Generated by create next app",
};

export default function RootLayout({ children }) {
  return (
    <html
      lang="en"
      className={`${geistSans.variable} ${geistMono.variable} h-full antialiased`}
    >
      <body className="min-h-full flex flex-col">
        <Navbar />
        {children}
      </body>
    </html>
  );
}
```

<br>

`app/page.js`
```javascript
export default function Home() {

    <div className="flex flex-col flex-1 items-center justify-center bg-zinc-50 font-sans dark:bg-black">

    </div>
  </>)
}
```

This is how our apps should be. Reload only when needed.  
If you notice the example, the title also changes without reloading.  
So, ditch this `<a>` tag and start using `<Link>` tag.

## `Script` Component - The better version of `<script>` tag

If we want to include a JS script in our app, we use the traditional `<script src=" ">` tag.  
But the browser stops everything and downloads that JS script file, then continues.  
Your page freezes while that happens, which is bad for performance.

**Next.js** has its own script tag `<Script>` which is a loooooooot better than the `<script>` tag.

Let me use the above example and demonstrate this tag:

`app/page.js`
```javascript
import Script from "next/script"

export default function Home() {

  return (<>
    <div className="flex flex-col flex-1 items-center justify-center bg-zinc-50 font-sans dark:bg-black">
      <h1 className="text-3xl">Hey, I am HOME page!</h1>
    </div>
    
    <Script src="/script.js"></Script>
  </>)
}
```

`public/script.js`
```javascript
const start = Date.now()
while (Date.now() - start < 3000) {}

alert("Hello, there!")
```
This script is made to be loaded slow, on purpose, to simulate a heavy script.

In plain HTML, when you use `<script src=" ">` to load JS files, you have zero control over when it loads, which can drop your page performance.  
**Next.js** `<Script>` tag has a prop called `strategy` which lets you control when the script loads.

It's very simple to use:

A user comes to your site. What's more important?

- The user must see the page fast? **Use `lazyOnload`**
  ```javascript
  <Script src="/script.js" strategy="lazyOnload"></Script>
  ```
- The script must run before user sees the page? **Use `beforeInteractive`**
  ```javascript
  <Script src="/script.js" strategy="beforeInteractive"></Script>
  ```
- Somewhere in between these two above? (in most cases) **Use `afterInteractive`**
  ```javascript
  <Script src="/script.js" strategy="afterInteractive"></Script>
  ```

This is the most important thing you should know about **Next.js** `<Script>` component.  
And now you know it!

## `Image` Component - The better version of `<img>` tag

This component does the optimization work a lot better than the `<img>` tag.  
There are some important props of this component you mustn't skip at all:

<br>

- `priority` prop

By default, every `<Image>` is lazy loaded, which means it only starts downloading when the user scrolls near it. That's great for keeping the app fast, but for images which are immediately visible when the page loads, lazy loading hurts the performance. The user sees empty space, which ain't good. 

`priority` fixes that. When you add `priority` prop to an `<Image>`, it gets downloaded before even the user discovers it.
```javascript
<Image src={GOT} alt="" priority className="w-200"></Image>
```

**NOTE:** Don't add `priority` prop to every image in your app.  
That just defeats the whole purpose of using it.  
Use it only on **Largest Contentful Paint (LCP)** images, which are important images.

*`priority` is been deprecated since version 16 of **Next.js**, so avoid using it.*  
*Instead, we can use `preload` prop, which is the official replacement of `priority`.*  
*`preload` is **false** by default. Make it **true** so the image preloads.*

<br>

- `placeholder` prop

Yes, you heard it right?  
In **Next.js**, even images can have placeholders.

```javascript
<Image src={GOT} alt="" placeholder="blur" className="w-200"></Image>
```

While the image is loading, the placeholder will be a blurry version of that image.  
Instead of being empty, and suddenly the image appearing, this is a lot smoother and better.

<br>

- `quality` prop

This controls how much **Next.js** compresses the image.  
Takes a number from 1 to 100. **Default is 75.**

```javascript
<Image src={GOT} alt="" width={1200} height={600} quality={90} />
```

Higher the `quality` value, the better the image. The size of the image will be big.  
Lower the `quality` value, the image will be slightly degraded. The size of the image will be smaller.
